Overview

In Master Combat Core, the Entity system is the foundation of all combat logic.
Damage, conditions, attributes, buffs, debuffs, AI decisions — everything operates on Entities, not GameObjects.

This page explains what an Entity is, how it is managed, and how it fits into graphs.

What Is an Entity?​

An Entity is a lightweight data container that represents a combat participant.

Examples:

  • Player
  • Enemy
  • Summoned unit
  • Projectile
  • Temporary graph instance entity

An Entity is not a GameObject, but it can be linked to one through an EntityComponent.

Each Entity has:

  • A unique ID (uid)
  • A set of Attributes
  • Runtime state used by graphs and systems

EntityComponent (Unity Side)​

To connect a GameObject to the combat system, attach an:

EntityComponent

This component:

  • Registers the GameObject as an Entity
  • Exposes attributes to graphs
  • Allows physics detection to resolve targets

EntityManager​

The EntityManager is a global registry that stores and resolves Entities.

Responsibilities:

  • Create and register Entities
  • Resolve Entities by uid
  • Track runtime Entities
  • Provide access to player, targets, and instances

Built-in Entity Types​

Player Entity​

  • Always registered with uid: player

Dealer Entity​

  • Assigned dynamically per graph instance
graphInstance.SetDealerEntity(entity);

Target Entity​

graphInstance.SetTargetEntity(entity);

Specified Entity​

  • Explicit lookup by uid

Instance Entity​

  • Temporary entity owned by a graph instance

Entity Node (Graph)​

The Entity Node outputs an Entity reference (Blue port) used by:

  • Condition nodes
  • Action nodes
  • Attribute variables

Attributes on Entities​

Attributes are numeric values such as:

  • Health
  • Attack
  • Defense
  • CritRate
  • MoveSpeed

They are read and modified by graphs at runtime.

Entity Lifecycle​

  1. GameObject spawns
  2. EntityComponent registers entity
  3. Graphs operate on entity
  4. Entity removed on destroy

Why This Matters​

The Entity system enables:

  • Fully data-driven combat
  • Reusable graphs
  • Clean scaling for AI, loot, and difficulty
